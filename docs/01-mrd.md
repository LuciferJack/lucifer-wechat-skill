# MRD: macOS 微信消息发送 Skill v2.0

## 1. 背景与问题

### 1.1 现状

目前有两套方案，均不稳定：

| 方案 | 技术路线 | 失败模式 |
|------|----------|----------|
| wechat-mcp | Python Accessibility API (AXUIElement) | 搜索框定位失败：`find_search_field` 按 `title="Search"` 查找 AXTextArea，中文系统下 title 可能不是 "Search" |
| wechat-mac skill | AppleScript 键盘模拟 (Cmd+F → paste → Enter×2) | Enter×2 触发"搜一搜"网页搜索，而非选中本地联系人；群聊需猜 Down 键次数 |

### 1.2 根因分析

**wechat-mcp 的核心问题**：
- 搜索框查找条件 `role == kAXTextAreaRole and title == "Search"` 在中文系统 / 新版微信下匹配不到
- 但其后续逻辑（`open_chat_for_contact` → 搜索列表 DFS → 精确匹配点击 → `find_input_field` → 直接写入 → 发送）是**架构正确**的

**wechat-mac skill 的核心问题**：
- 依赖固定键盘序列（Enter 次数、Down 次数），无法适应搜索结果动态变化
- 截图验证依赖 AI 识图，增加延迟和不确定性
- 搜索打开的是全局搜索（含网页搜索），不是侧边栏本地搜索

## 2. 目标

| 优先级 | 目标 | 衡量标准 |
|--------|------|----------|
| P0 | 稳定发送文本消息到任意联系人/群聊 | 成功率 ≥ 95%（连续 20 次测试） |
| P0 | 零误发 | 发送前通过 AX API 读取当前聊天标题，与目标精确匹配 |
| P1 | 快速（< 8 秒完成搜索+验证+发送） | 从调用到发送完成的端到端时间 |
| P1 | 清晰的错误反馈 | 失败时返回可操作的错误信息，而非静默失败 |
| P2 | 发送后可验证 | 通过 wechat-cli 确认消息到达 |

## 3. 用户场景

### 3.1 核心场景

1. **发消息给联系人**：`"给文件传输助手发：测试消息"`
2. **发消息给群聊**：`"给 xxx 群发：今天的会议取消了"`
3. **批量发送**：`"给 A、B、C 分别发送：xxx"`

### 3.2 边界场景

- 联系人不在最近会话列表（需搜索）
- 同名联系人（需用户消歧）
- 群名含特殊字符（括号、emoji）
- 微信未登录/未启动
- 屏幕锁定状态

## 4. 技术方案概要

### 4.1 核心架构：Accessibility API + 多策略导航

基于 wechat-mcp 的 `wechat_accessibility.py` 架构，做三层改进：

```
┌─────────────────────────────────────────────┐
│                  Skill 入口                  │
│  安全检查 → 用户确认 → 发送 → 验证           │
└───────────────┬─────────────────────────────┘
                │
┌───────────────▼─────────────────────────────┐
│             Chat Navigator                   │
│  Strategy 1: 会话列表直接点击                │
│  Strategy 2: AX 搜索框搜索 + 结果精确匹配    │
│  Strategy 3: AppleScript Cmd+F 降级          │
└───────────────┬─────────────────────────────┘
                │
┌───────────────▼─────────────────────────────┐
│            Target Verifier                   │
│  读取 AX 聊天标题 (big_title_line_h_view)    │
│  精确匹配 → 通过 / 不匹配 → 中止             │
└───────────────┬─────────────────────────────┘
                │
┌───────────────▼─────────────────────────────┐
│            Message Sender                    │
│  AX 定位输入框 (chat_input_field)            │
│  SetAttributeValue → Press Return            │
└─────────────────────────────────────────────┘
```

### 4.2 关键改进点

| 改进 | 说明 |
|------|------|
| **搜索框多条件定位** | 不再只按 `title="Search"` 查找，增加 `identifier` 匹配、role+位置启发式匹配 |
| **会话列表优先** | 先在侧边栏会话列表找（`session_item_*`），找到直接点击，省去搜索 |
| **程序化验证** | 用 AX API 读 `big_title_line_h_view` 的值来验证目标，替代截图+AI 识别 |
| **窗口状态预检** | 发送前确认微信已激活、主窗口可见、非最小化 |
| **输入框直写** | 用 `AXUIElementSetAttributeValue` 直接写入消息，不依赖剪贴板 |

### 4.3 与现有方案的关系

- **替换** wechat-mac skill 的发送能力（保留查看能力不变）
- **不替换** wechat-mcp（那是独立的 MCP server，可共存）
- 复用 wechat-mcp 的 AX 工具函数（`dfs`、`click_element_center`、`axvalue_to_point` 等）

## 5. 安全规则（继承现有）

- 隐私内容拦截（密码、银行卡、账号、密钥、隐私信息）
- 发送前必须用户确认
- 发送后通过 wechat-cli 验证

## 6. 范围

### In Scope
- 文本消息发送（联系人 + 群聊）
- 窗口状态管理（激活、防锁屏）
- 错误处理与用户反馈

### Out of Scope
- 图片/文件/语音发送
- 自动回复/机器人
- 消息撤回
- 多账号支持
